## Project Purpose
React Native / Expo mobile app that displays real-time energy data from a Shelly EM/Pro energy meter device installed at a residential property. The Shelly device measures:
- **Solar production**: electricity generated by rooftop solar panels
- **House consumption**: electricity used by home appliances (TV, AC, etc.)

The app helps the homeowner understand their energy balance at a glance — whether they are producing more solar energy than they consume (surplus/export) or drawing more from the grid than their panels provide (deficit/import). Targets iOS, Android, and Web.

## Development Commands

```bash
npm start          # Start Expo dev server (scan QR or press a/i/w)
npm run android    # Launch on Android emulator / connected device
npm run ios        # Launch on iOS simulator / connected device
npm run web        # Launch in browser
```

No build, lint, or test scripts are configured yet.

## Architecture

**Entry flow**: `index.ts` → registers `App.tsx` as the root component via `expo`'s `registerRootComponent`.

**Stack**:
- React 19 + React Native 0.81 + Expo 54 (new architecture enabled)
- TypeScript 5 in strict mode (`tsconfig.json` extends Expo base config)
- React Navigation v7 (`@react-navigation/native` + `native-stack`) installed but not yet wired up in `App.tsx`
- `react-native-safe-area-context` and `react-native-screens` as navigation peer deps

**Current state**: `App.tsx` is a placeholder. Navigation and Shelly device communication have not been implemented yet.
